Фінальна Архітектура (JWT + Services)

Спроєктовано на основі вашого референсу та схеми БД

Технологічний стек
  • Framework: Laravel 12.0
  • Auth: JWT (php-open-source-saver/jwt-auth)
  • Database: PostgreSQL 17
  • Documentation: L5-Swagger
  • Pattern: Controller -> Service -> Model

Структура БД (Server-side)

Користувачі та Ролі

Таблиця Users з полем role (admin/user).

Пристрої (PCs)

Таблиця PCs (UniqueID, user_id, name). Зв'язок один-до-багатьох з користувачем.

Логи (Activity)

Таблиці Processes та Schedules. Кожен запис має pc_id.

API Ендпоінти

// Auth
POST /api/v1/auth/login     -> {access_token}
POST /api/v1/auth/register  -> Створення акаунта

// Sync (Batch)
POST /api/v1/sync/processes -> Пакетна синхронізація вікон
POST /api/v1/sync/schedules -> Синхронізація статусів ПК

// Web (Blade)
GET /dashboard              -> Перегляд даних (фільтрація за ПК)
GET /admin/users            -> (Тільки для Admin) Керування користувачами

Чому це підходить

  • Консистентність: Використовуються ті ж бібліотеки, що і у вашому проєкті (JWT, Swagger).
  • Автоматизація: ПК створюється автоматично при першому надсиланні даних з новим UniqueID.
  • Docker: Використання php:8.4-apache та postgres:17 як у референсі.
