Архітектура REST API та Синхронізація

Пропонована структура ендпоінтів на основі вашої схеми БД

POST /api/v1/sync/processes

Синхронізація активності вікон. Приймає масив об'єктів: UniqueID, start, name, window, duration.

POST /api/v1/sync/schedules

Запис статусів ПК (Ввімкнено/Вимкнено). Передає timestamp та статус.

POST /api/v1/pc/register

Реєстрація нового ПК в системі. Повертає API Key для подальшої авторизації.

Логіка автентифікації (API Key)

Кожен запит від десктопного застосунку повинен містити заголовок:
X-API-KEY: [ваш_ключ]

Переваги обраного підходу

  • Проста реалізація на стороні клієнта (WinForms).
  • Надійна прив'язка даних до конкретного ПК.
  • Можливість легко анулювати доступ для окремого пристрою.
